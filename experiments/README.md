# Experiments Notebook

This folder is the research notebook and index for serious, pre-declared experiments in this repo. Each experiment is a directory named like `NNN-short-title/` and must contain its own `README.md`.

## What Belongs Here

- Pre-declared experiment folders with clear questions or hypotheses.
- A per-experiment `README.md` explaining the investigation and describing the files in that folder.
- Setup details, commands, configs, dataset/model/checkpoint versions, prompts, or approach.
- Results and evidence, summarized with links to artifacts when needed.
- Conclusions, decisions, impact on future work, and follow-up questions.

## What Does Not Belong Here

- Scratch notes, daily logs, vague ideas, or ordinary implementation notes.
- Failed one-off commands with no analyzed conclusion.
- Raw outputs or large artifacts; link to them instead.
- Work-in-progress notes that have not reached a clear experimental conclusion.
- Agent-created experiment folders or log entries that were not explicitly requested or approved by the user.

## How To Use This Notebook

1. Read this index before proposing a new research approach so past serious attempts inform the plan.
2. Determine and declare the experiment before the project/run starts, normally by explicit user command or an approved plan.
3. Create a folder named `NNN-short-title/` from `000-template/` only when the user explicitly commands or approves the experiment.
4. Keep the experiment's `README.md` updated with the question, scope, file list, setup, procedure, evidence, conclusion, and impact.
5. Agents may consult this index when considering new approaches, but must not create a new experiment folder or index/log entry on their own without explicit user instruction or approval.
