# Reasoning-Pruning PT Dataset Objective

This repo creates local transition reasoning-pruning (PT) data for reasoning tasks. The runner generates model reasoning, asks a live LiteLLM decision model for a safely removable contiguous span, verifies the shortened context, and saves accepted training records.

Run the active Gemma4 workflow through Hugging Face Jobs; the job uses the normal config runner:

```bash
uv run --extra hf --extra gemma4 python scripts/create_pruning_dataset.py --config config/bbh-logical-deduction-gemma4-hf-preview.toml
```

Dataset-specific runs use the same CLI/config pattern with another config under `config/*.toml`. Prompts, source settings, model settings, output paths, quality gates, and upload settings live in config; HF Jobs supplies the execution environment.

Accepted records use the local transition shape: `input_x` is the question/current context plus the useful prefix, `target_y` is the next useful step after pruning, and metadata records the removed span, quality checks, source task, model settings, and `format_version`.
